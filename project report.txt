# 📊 Customer Churn Analysis – Full Project Report

## 📌 1. Introduction

Customer churn is one of the most critical challenges faced by subscription-based businesses such as telecom companies. High churn leads to revenue loss and increased customer acquisition costs.

This project aims to analyze customer churn behavior, identify key influencing factors, and provide actionable insights using data analysis, visualization, and machine learning.

---

## 🎯 2. Objectives

* Analyze customer data to identify churn patterns
* Perform data cleaning and preprocessing
* Conduct exploratory data analysis (EDA)
* Build a machine learning model to predict churn
* Create an interactive dashboard for business insights

---

## 📂 3. Dataset Description

The dataset contains customer information including:

* Customer ID
* Gender
* Senior Citizen status
* Contract type
* Monthly Charges
* Total Charges
* Tenure
* Payment Method
* Churn (Target variable: Yes/No)

---

## 🧹 4. Data Cleaning (Excel)

Data cleaning was performed using **Microsoft Excel**:

* Removed missing values
* Fixed inconsistent formats
* Removed duplicates
* Standardized categorical values (e.g., Yes/No)
* Corrected spacing and text errors

👉 Clean data was then used for analysis and modeling

---

## 📊 5. Exploratory Data Analysis (EDA)

EDA was performed using Python (Pandas, Matplotlib, Seaborn):

### Key Observations:

* **Contract Type:**

  * Month-to-month customers have the highest churn
  * Long-term contracts reduce churn

* **Monthly Charges:**

  * Higher charges correlate with higher churn

* **Tenure:**

  * Customers with low tenure are more likely to churn

* **Payment Method:**

  * Certain methods (e.g., electronic check) show higher churn

* **Demographics:**

  * Gender and senior citizen status have less impact

---

## 🤖 6. Machine Learning Model

A classification model was built to predict churn:

* Algorithm used: Logistic Regression (or similar)
* Target variable: Churn (Yes/No)
* Features: Contract, Charges, Tenure, etc.

### Outcome:

* Model successfully identifies high-risk customers
* Can be used for proactive retention strategies

---

## 📊 7. Power BI Dashboard

An interactive dashboard was created to visualize insights:

### Key Features:

* KPI Cards:

  * Total Customers
  * Churn Customers
  * Churn Rate

* Visualizations:

  * Contract vs Churn
  * Gender & Senior Citizen analysis
  * Monthly Charges distribution
  * Tenure vs Churn
  * Payment Method vs Churn

---

## 🔍 8. Key Business Insights

* Customers with **month-to-month contracts** churn the most
* **High monthly charges** increase churn probability
* **New customers (low tenure)** are at high risk
* **Payment methods** influence churn behavior

---

## 🚀 9. Recommendations (Product Perspective)

* Promote long-term contracts with discounts
* Improve onboarding experience for new users
* Introduce pricing flexibility
* Optimize payment experience (auto-pay incentives)
* Target high-risk customers with retention campaigns
